# Internacionalizacao

O projeto tem uma base leve de i18n em `assets/js/i18n.js`.

## Idiomas iniciais

- `assets/i18n/pt-BR.json`
- `assets/i18n/en-US.json`
- `assets/i18n/es-ES.json`

## Como usar

1. Mantenha o texto em portugues no HTML como fallback.
2. Adicione `data-i18n="chave"` para textos.
3. Adicione `data-i18n-placeholder="chave"` para placeholders.
4. Adicione `data-i18n-aria-label="chave"` para labels de acessibilidade.

Exemplo:

```html
<h1 data-i18n="settings.hero.title">Configuracoes da conta</h1>
```

O idioma e detectado por `localStorage["educaria:i18n:language"]` ou por `navigator.language`. Para trocar manualmente:

```js
setEducariaLanguage("en-US");
```

Na pagina de configuracoes, o campo `data-settings-language` ja usa essa funcao e persiste a escolha.

Se o JSON nao carregar, a pagina permanece com o texto original do HTML.

## Checklist para novas paginas

1. Incluir `assets/js/i18n.js` antes do script especifico da pagina.
2. Adicionar chaves no HTML apenas em textos estaticos.
3. Para textos gerados por JS, usar `window.educariaTranslate("chave") || "fallback"`.
4. Atualizar os tres dicionarios no mesmo PR.
5. Rodar parse JSON antes de commitar:

```powershell
node -e "const fs=require('fs'); for (const file of ['assets/i18n/pt-BR.json','assets/i18n/en-US.json','assets/i18n/es-ES.json']) JSON.parse(fs.readFileSync(file,'utf8'));"
```
